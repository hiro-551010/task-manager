# task ドメインルール（不変条件・バリデーション）

## Title

| ルール | 内容 |
|---|---|
| 空文字禁止 | タイトルは1文字以上必須 |
| 最大長 | 100文字以内 |

## TaskStatus 遷移

| 現在 | 許可する遷移先 |
|---|---|
| `todo` | `in_progress` のみ |
| `in_progress` | `todo` / `done` |
| `done` | なし（終端状態） |

- 許可されていない遷移を試みた場合は `DomainError` を返す。

## DueDate

| ルール | 内容 |
|---|---|
| 過去日付禁止 | 設定・更新時点より過去の日付は不可 |
| 省略可能 | null を許容する |

## Task 作成時

- `status` の初期値は必ず `todo`。
- `id` は作成時に自動生成する（外部から指定不可）。
- `ownerId` は作成時に必須。後から変更不可。

## 所有権

| ルール | 内容 |
|---|---|
| 参照制限 | ログインユーザー自身が所有するタスクのみ取得・操作できる |
| 越権操作禁止 | 他ユーザーのタスクへの操作は `ApplicationError`（403相当） |
